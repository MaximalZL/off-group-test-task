# Тестовое задание: модуль `offgroup.bonuses`

Локальный модуль для Bitrix24, реализующий систему бонусов сотрудников.

## Постановка задачи

Необходимо было разработать локальный модуль для системы на базе 1С-Битрикс24, который реализует систему бонусов для сотрудников.

Требования тестового задания:

- создать ORM-сущность D7 для хранения бонусов сотрудников
- добавить в левое меню Bitrix24 пункт `Мои бонусы`
- сделать страницу со списком бонусов текущего пользователя с использованием `main.ui.grid`
- выводить в таблице колонки `Сотрудник`, `Дата начисления`, `Сумма бонуса`, `Причина`
- реализовать обработчик события CRM, который при закрытии сделки начисляет ответственному менеджеру `1%` от суммы сделки
- реализовать REST-метод `offgroup.bonuses.gettotal`
- метод должен принимать параметр `employee_id`
- метод должен возвращать объект с полем `total`, содержащим общую сумму бонусов сотрудника

Ниже описано, как эти требования реализованы в модуле.

Реализовано:

- D7 ORM-сущность для хранения бонусов сотрудников
- пункт меню `Мои бонусы`
- страница со списком бонусов текущего пользователя на `main.ui.grid`
- автоматическое начисление бонуса при успешном закрытии сделки
- REST-метод `offgroup.bonuses.gettotal` для внешних интеграций

## Структура

- `local/modules/offgroup.bonuses` - корень модуля
- `local/modules/offgroup.bonuses/lib/BonusTable.php` - ORM-сущность бонусов
- `local/modules/offgroup.bonuses/lib/Service/BonusService.php` - бизнес-логика начисления и расчета общей суммы
- `local/modules/offgroup.bonuses/lib/Integration/CrmDealBonusHandler.php` - обработчик CRM-события
- `local/modules/offgroup.bonuses/lib/Integration/MenuBuilder.php` - добавление пункта меню
- `local/modules/offgroup.bonuses/lib/Rest/BonusRestService.php` - регистрация и обработка REST-метода
- `local/modules/offgroup.bonuses/install/components/offgroup/bonuses.list` - компонент списка бонусов
- `local/modules/offgroup.bonuses/install/public/offgroup/bonuses/index.php` - публичная страница `/offgroup/bonuses/`

## Что делает модуль

### 1. Хранение данных

Модуль создает таблицу `offgroup_bonuses` через D7 ORM.

Ключевые поля:

- `ID`
- `EMPLOYEE_ID`
- `ACCRUED_AT`
- `BONUS_AMOUNT`
- `REASON`
- `SOURCE_ENTITY_TYPE`
- `SOURCE_ENTITY_ID`
- `CREATED_AT`

Поля `SOURCE_ENTITY_TYPE` и `SOURCE_ENTITY_ID` используются, чтобы не начислять бонус повторно за одну и ту же сущность, например за одну и ту же сделку.

### 2. Пользовательский интерфейс

В левое меню добавляется пункт `Мои бонусы`.

При установке модуля в публичной части сайта создается страница:

- `/offgroup/bonuses/`

По этому адресу открывается страница со списком бонусов текущего пользователя. Таблица построена на `main.ui.grid` и выводит колонки:

- Сотрудник
- Дата начисления
- Сумма бонуса
- Причина

### 3. Автоматическое начисление

На событие `crm:OnAfterCrmDealUpdate` подписан обработчик `Offgroup\Bonuses\Integration\CrmDealBonusHandler::onAfterDealUpdate()`.

Далее логика передается в `Offgroup\Bonuses\Service\BonusService::accrueForDeal()`:

- загружается сделка
- проверяется, что стадия сделки имеет семантику `SUCCESS`
- определяется ответственный сотрудник
- рассчитывается бонус как `1%` от `OPPORTUNITY`
- в таблицу бонусов добавляется запись с причиной вида `Бонус за закрытую сделку №123`

### 4. REST API

Модуль регистрирует REST-метод:

- `offgroup.bonuses.gettotal`

Параметры:

- `employee_id` - ID сотрудника

Ответ:

```json
{
  "total": 1500
}
```

Важно: проверка авторизации не дублируется внутри метода через `CurrentUser`, поскольку внешний вызов проходит стандартную авторизацию Bitrix24 REST на уровне вебхука или приложения. В самом методе валидируется входной параметр `employee_id`, после чего вызывается сервис расчета суммы бонусов.

Пример вызова:

```http
POST /rest/1/webhook_key/offgroup.bonuses.gettotal.json
employee_id=15
```

## Установка

1. Скопировать папку `local/modules/offgroup.bonuses` в проект Bitrix24.
2. Убедиться, что установлены модули `crm` и `rest`.
3. Установить модуль через административный интерфейс Bitrix24.
4. После установки открыть страницу `/offgroup/bonuses/`.

Во время установки модуль:

- регистрирует себя в системе
- создает таблицу `offgroup_bonuses`
- копирует публичную страницу и компонент
- регистрирует обработчики событий CRM, меню и REST

После установки публичная страница доступна по адресу `/offgroup/bonuses/`.

## Технические замечания

- Настройки модуля через `options.php` не реализованы, процент бонуса и условия начисления сейчас заданы в коде.
- Логика начисления привязана к успешной стадии сделки с семантикой `SUCCESS`.
- Защита от повторного начисления реализована на уровне проверки существующей записи с той же связкой `EMPLOYEE_ID + SOURCE_ENTITY_TYPE + SOURCE_ENTITY_ID`.

## Проверка

В рамках подготовки решения выполнена проверка PHP-синтаксиса для всех файлов модуля командой `php -l`.

Полноценный запуск внутри конкретного экземпляра Bitrix24 в этой среде не выполнялся, поэтому интеграционные детали, зависящие от конфигурации портала, требуют проверки уже на установленной системе.
